# スキル改善ワークフロー

## 概要

> **「AIに手順を追わせるのではなく、専門家のように"考えさせる"」**
> — @boringmarketer

スキルは一度書いて終わりではない。実際の失敗から学び、継続的に改善する。

---

## 10ステッププロセス

1. **UNDERSTAND** → 何のスキル？何の問題？
2. **EXPLORE** → 既存スキルだけで実行し、失敗・不足箇所を観察
3. **RESEARCH** → ドメインを深堀り（Web検索、ドキュメント）
4. **SYNTHESIZE** → リサーチから原則を抽出
5. **DRAFT** → 初期スキルを書く
6. **SELF-CRITIQUE** → 品質基準でレビュー
7. **ITERATE** → フィードバックで改善
8. **TEST** → 実シナリオでテスト
9. **FINALIZE** → 最適な構造に整理

---

## 4つの原則

| 原則 | 意味 |
|------|------|
| **Expertise Transfer** | 手順ではなく、専門家の思考を移植 |
| **Flow, Not Friction** | 中間ドキュメントではなく、出力を生む |
| **Voice Matches Domain** | ドキュメント調ではなく、実務者の声で |
| **Focused Beats Comprehensive** | 容赦なく絞る。全セクションに存在理由を |

## 実践ガイド

### 新スキル作成時

```
1. まず既存スキルだけでタスクを実行
2. 失敗・非効率な箇所を記録
3. その領域のベストプラクティスを調査
4. 原則を抽出してSKILL.mdをドラフト
5. 再テストして効果を確認
```

### 既存スキル改善時

```
1. 繰り返し発生する問題を特定
2. 問題の根本原因を分析
3. 改善案を提案（ユーザー承認）
4. スキルを更新
5. 効果を検証
```

---

## 注意事項

- **完全自動化は避ける** → 提案 → 承認 → 適用のセミオート運用
- **スキルの肥大化に注意** → 定期的に不要なセクションを削除
- **1スキル1目的** → 複数の目的を混ぜない
